## ğŸ› ï¸ Implementation Plan

1.  **Update Config**:
    *   Create a new `BuildingCostConfig` class that maps each `templateId` (e.g., 'wall_basic') to its `ResourceCost`.
    *   Integrate this into the app's startup or `BaseBuildingViewModel`.

2.  **Resource Check (Pre-Build)**:
    *   In `BaseBuildingViewModel.placeItem`, BEFORE placing, lookup the cost.
    *   Check if `CastleGroundsViewModel.castle` has enough resources.
    *   If NOT, show a "Not enough resources!" error toast/dialog.

3.  **Resource Deduction (On-Build)**:
    *   If resources are sufficient, call a new API/method in `CastleGroundsViewModel` to spend the resources.
    *   Optimistically update the UI (subtract local counts).
    *   Send the placement + deduction to the backend (or separate calls if needed).

4.  **UI Updates**:
    *   In the "Build Menu" (Inventory), show the COST of each item under its icon.
    *   Gray out items that are too expensive.
